# OS Projects Guidelines for SWU@Summer Term

## Setups
Basically, we follow the project settings (some are modified) of [OSTEP](https://pages.cs.wisc.edu/~remzi/OSTEP/). 
You need to prepare the following setups. 

- Linux environment (WSL or VM)
- C toolchain
- make/cmake
- Source Code: [https://github.com/remzi-arpacidusseau/ostep-projects](https://github.com/qyliu-hkust/ostep-projects)

## Requirements
What should you submit? 

- Source code that **YOU** write (e.g., `wcat.c`)
- A validation report generated by script `./test-xx.sh` under each sub-project folder
- A technical project report written in English on the implementation details and discussions on experiment results

Note: the report template can be found under this folder

## Some General Instructions

- Ask **Google** first when you meet some questions
- LLMs are allowed, but make sure you **FULLY UNDERSTAND** AI-generated content
- Always check [Linux manual](https://man7.org/linux/man-pages/index.html) when you are not familiar with some functions or utilities
- **NO BARGAIN AND CHEATING**
